# LAI_scheduler_update

LAI AP injection alarm scheduler development

1. **Procedure detail**: 주요 작업 내용
   1. 기존에 만들었던 scheduler 파일의 코드를 읽고 수정한다.
   2. 캘린더에서 각 행과 열의 크기를 바꾸지 말고 전체 사이즈에 비례해서 변경되게 한다.
   3. 캘린더를 상용 프로그램 수준으로 더 예쁘게 디자인한다.
   4. 오늘은 다른 날짜와 색깔을 구분되게 만든다.
   5. 캘린더에 각 날짜에 환자명은 블록으로 표시된다. 예약자는 옅은 주황색 블록을 사용하고, 완료된 사람은 연두색 블록으로 표현한다.
   6. 만약 이전에 입력된 내용을 삭제하거나 수정하면, 예약된 내용도 삭제되거나 수정되도록 만든다.
   7. 만약 예약된 당일 환자가 내원해서 예약된대로 주사를 맞게 되면, 예약을 삭제하고 다시 입력을 하는 것이 아니라, 단순히 예약 내용을 더블 클릭해서 들어가면 입력 당시와 동일한 인터페이스에 기존 예약된 내용이 그대로 표시되고, '완료' 버튼을 누르면 예약 내용이 사라지고, 변경 사항이 입력되며, 다음 예약도 입력된다.
   8. 캘린더 각 날짜에는 예약자의 이름이 하나의 블록처럼 표시되고, 블록을 이동해서 예약일을 변경할 수 있게 한다. 만약, 3월 20일에 예약된 환자 홍길동이 3월 18일에 내원했다면, 3월 20일에 있던 홍길동의 블록을 3월 18일로 이동시키면, 3월 20일의 블록이 3월 18일로 이동되고, 이 내용을 더블클릭해서 수정 사항을 변경한 후 완료버튼 누르면 예약 내용은 사라지고, 새로운 입력이 표시된다.
   9. 좌측 상단에 Otsuka의 로고를 추가할 것이다.
   10. 우측 상단에 Abilify Maintena의 이미지를 추가할 것이다.
   11. 내가 사용할 로고와 이미지는 data_image 폴더에 저장해 둘 것이다.
   12. Abilify Maintena의 이미지는 다소 부드럽게 변경하고, 테두리 없이 배경과 그라데이션이 되게 처리하라.
   13. 팝업 알람은 기본적으로 프로그램을 시작할 때 켜지도록 한다.
   14. 팝업 알림은 오늘, 내일, 모레 3개의 창을 각각 다르게 띄우며, 오늘은 분홍색 계열, 내일은 주황색 계열, 모레는 하늘색 계열을 사용하라.
   15. 알람 강제 테스트 기능은 없앨 것이다.
   16. default alarm 이외에 추가로 알람을 설정할 수 있게 하고, 이 경우 알람은 오전 7시로 설정된다. 옵션은 1일 전, 2일 전 두 가지가 있다.
   17. 캘린더에 설정된 예약자와 내용을 자신의 스마트폰에서도 볼 수 있게 연동되도록 만든다. 그러려면 프로그램에 자신의 구글 계정을 입력하는 칸을 만들고, 구글 캘린더와 연동되도록 한다.
   18. 최종적으로는 실행파일을 만들어서 배포할 것이다.
